# ⏱️ 番茄鐘記錄整合測試指南

## 🎯 功能說明

現在每次完成或中斷番茄鐘時，系統會自動記錄到 SQLite 資料庫，包括：
- 📅 開始和結束時間
- ⏱️ 實際時長（分鐘）
- ✅ 完成狀態（完成/未完成）
- 🏷️ 會話類型（專注/休息）
- 🔗 關聯的任務（如果有選擇任務）

---

## 🔄 記錄邏輯

### 情況 1：完成番茄鐘 ✅
**觸發條件：** 計時器倒數到 0
**記錄內容：**
- `completed = true`
- `duration = 實際時長`（通常是 25 分鐘）
- `end_time = 完成時間`

**終端機日誌：**
```
⏱️ [TIMER] 開始新的番茄鐘會話 (ID: xxxxx)
✅ [TIMER] 番茄鐘會話已記錄到資料庫
   - 任務: [任務名稱]
   - 時長: 25 分鐘
   - 完成: true
⏱️ [DB] 插入番茄鐘會話: 任務ID=xxxxx, 時長=25分鐘, 完成=true, 類型=focus
✅ [DB] 番茄鐘會話插入成功 (ID: x)
```

---

### 情況 2：中途停止 ⏹️
**觸發條件：** 點擊「停止」按鈕
**記錄內容：**
- `completed = false`
- `duration = 已經過時長`（例如：15 分鐘）
- `end_time = 停止時間`

**終端機日誌：**
```
⏱️ [TIMER] 開始新的番茄鐘會話 (ID: xxxxx)
✅ [TIMER] 番茄鐘會話已記錄到資料庫
   - 任務: [任務名稱]
   - 時長: 15 分鐘
   - 完成: false
⏱️ [DB] 插入番茄鐘會話: 任務ID=xxxxx, 時長=15分鐘, 完成=false, 類型=focus
✅ [DB] 番茄鐘會話插入成功 (ID: x)
```

---

### 情況 3：暫停後繼續 ⏸️
**行為：** 暫停不會記錄會話，繼續計時使用同一個會話 ID
**記錄：** 只在最終完成或停止時記錄一次

---

### 情況 4：無關聯任務 📝
**觸發條件：** 未選擇任何任務時開始計時器
**記錄內容：**
- `task_id = null`
- 其他資訊正常記錄

**終端機日誌：**
```
✅ [TIMER] 番茄鐘會話已記錄到資料庫
   - 任務: 無關聯任務
   - 時長: 25 分鐘
   - 完成: true
```

## 🧪 測試步驟

### 測試 1：完成完整的番茄鐘 ✅

1. **準備：**
   - 在「任務」頁面創建一個測試任務
   - 點擊任務的 ⏱️ 計時器按鈕
   - 確認切換到計時器頁面

2. **執行：**
   - 點擊「開始」按鈕
   - 觀察終端機輸出：`⏱️ [TIMER] 開始新的番茄鐘會話`
   - 等待計時器倒數（可以在設定中調整為 1 分鐘測試）
   - 或者點擊「跳過」按鈕快速完成

3. **驗證：**
   - 終端機顯示：
     ```
     ✅ [TIMER] 番茄鐘會話已記錄到資料庫
        - 任務: [你的任務名稱]
        - 時長: 25 分鐘
        - 完成: true
     ⏱️ [DB] 插入番茄鐘會話...
     ✅ [DB] 番茄鐘會話插入成功
     ```
   - 計時器自動切換到休息模式
   - 任務的完成番茄鐘數量 +1

---

### 測試 2：中途停止番茄鐘 ⏹️

1. **準備：**
   - 選擇一個任務
   - 切換到計時器頁面

2. **執行：**
   - 點擊「開始」按鈕
   - 等待 5-10 秒（或更長）
   - 點擊「停止」按鈕

3. **驗證：**
   - 終端機顯示：
     ```
     ✅ [TIMER] 番茄鐘會話已記錄到資料庫
        - 任務: [你的任務名稱]
        - 時長: 0 分鐘（少於1分鐘會顯示0）
        - 完成: false
     ```
   - 計時器重置到 25:00
   - 任務的番茄鐘數量不變

---

### 測試 3：暫停和恢復 ⏸️

1. **準備：**
   - 選擇一個任務
   - 開始計時器

2. **執行：**
   - 點擊「暫停」按鈕
   - 等待幾秒
   - 點擊「繼續」按鈕
   - 點擊「停止」或等待完成

3. **驗證：**
   - 暫停時不會記錄會話
   - 只在最終停止/完成時記錄一次
   - 記錄的時長 = 實際經過的時間（不包括暫停時間）

---

### 測試 4：無關聯任務 📝

1. **準備：**
   - 直接進入計時器頁面
   - 不選擇任何任務

2. **執行：**
   - 開始計時器
   - 等待完成或點擊停止

3. **驗證：**
   - 終端機顯示：
     ```
     ✅ [TIMER] 番茄鐘會話已記錄到資料庫
        - 任務: 無關聯任務
        - 時長: 25 分鐘
        - 完成: true
     ```
   - `task_id = null` 在資料庫中

---

### 測試 5：切換模式 🔄

1. **準備：**
   - 開始一個番茄鐘

2. **執行：**
   - 計時進行到一半時
   - 手動切換到「專注」或「休息」模式

3. **驗證：**
   - 如果有進行中的專注會話，會先記錄為未完成
   - 切換後開始新的會話

---

### 測試 6：連續多個番茄鐘 🔁

1. **準備：**
   - 選擇一個任務

2. **執行：**
   - 完成第 1 個番茄鐘（25 分鐘）
   - 跳過休息時間
   - 完成第 2 個番茄鐘
   - 跳過休息時間
   - 完成第 3 個番茄鐘

3. **驗證：**
   - 終端機顯示 3 次插入記錄
   - 每次都有獨立的會話 ID
   - 任務的完成番茄鐘數量 +3
   - 資料庫中有 3 條記錄

## 📊 資料庫查詢測試

你可以使用以下方式驗證資料：

#### 方法 2：應用重啟驗證
1. 完成幾個番茄鐘

2. 使用 DB Browser for SQLite 打開
3. 查詢 `pomodoro_sessions` 表

## 🔍 預期的資料庫結構

### pomodoro_sessions 表

| 欄位 | 類型 | 說明 | 範例 |
|------|------|------|------|
| id | INTEGER | 自動遞增主鍵 | 1, 2, 3... |
| task_id | TEXT | 關聯的任務 ID（可為空） | "abc-123-def-456" |
| start_time | TEXT | 開始時間（ISO 8601） | "2025-10-14T10:30:00.000" |
| end_time | TEXT | 結束時間（ISO 8601） | "2025-10-14T10:55:00.000" |
| duration | INTEGER | 時長（分鐘） | 25 |
| completed | INTEGER | 是否完成（1=是, 0=否） | 1 |
| session_type | TEXT | 會話類型 | "focus", "shortBreak", "longBreak" |

## ⚠️ 常見問題

### Q: 為什麼暫停時沒有記錄？
**A:** 暫停只是暫停計時，並不結束會話。只有在完成或停止時才會記錄。

### Q: 時長顯示 0 分鐘是正常的嗎？
**A:** 如果會話時長少於 1 分鐘，會顯示 0。這是因為我們用分鐘為單位記錄。

### Q: 如果沒有選擇任務會怎樣？
**A:** 會話仍然會記錄，但 `task_id` 為 `null`，終端機顯示「無關聯任務」。

### Q: 休息時間會記錄嗎？
**A:** 目前只記錄專注時間（focus）。休息時間的 `session_type` 會是 "shortBreak" 或 "longBreak"。
