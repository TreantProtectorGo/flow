# ⏱️ 番茄鐘記錄整合 - 快速測試

## 🚀 快速測試步驟

### 第一步：熱重載應用
在終端機按 `r` 或 `R` 重新載入應用

---

### 第二步：測試完整番茄鐘 ✅

1. **創建任務**
   - 在「任務」頁面點擊 ➕ 新增任務
   - 填寫任務資訊並保存

2. **開始計時**
   - 點擊任務的 ⏱️ 計時器按鈕
   - 確認切換到計時器頁面
   - 點擊「開始」按鈕

3. **觀察終端機**
   ```
   ⏱️ [TIMER] 開始新的番茄鐘會話 (ID: xxxxx)
   ```

4. **完成番茄鐘**
   - 方法 A：等待 25 分鐘（正常流程）
   - 方法 B：在設定中改為 1 分鐘（快速測試）
   - 方法 C：點擊「跳過」按鈕（最快）

5. **確認記錄**
   終端機應顯示：
   ```
   ✅ [TIMER] 番茄鐘會話已記錄到資料庫
      - 任務: [你的任務名稱]
      - 時長: 25 分鐘
      - 完成: true
   ⏱️ [DB] 插入番茄鐘會話: 任務ID=xxxxx, 時長=25分鐘, 完成=true, 類型=focus
   ✅ [DB] 番茄鐘會話插入成功 (ID: 1)
   ```

**檢查點：**
- ✅ 終端機顯示完整的記錄日誌
- ✅ 任務的番茄鐘數量 +1
- ✅ 計時器自動切換到休息模式

---

### 第三步：測試中途停止 ⏹️

1. **開始新的番茄鐘**
   - 再次點擊「開始」
   - 觀察終端機：`⏱️ [TIMER] 開始新的番茄鐘會話`

2. **等待一段時間**
   - 讓計時器運行 10-30 秒

3. **點擊停止按鈕**
   - 觀察終端機日誌

4. **確認記錄**
   終端機應顯示：
   ```
   ✅ [TIMER] 番茄鐘會話已記錄到資料庫
      - 任務: [你的任務名稱]
      - 時長: 0 分鐘（少於1分鐘）
      - 完成: false
   ⏱️ [DB] 插入番茄鐘會話: 任務ID=xxxxx, 時長=0分鐘, 完成=false, 類型=focus
   ✅ [DB] 番茄鐘會話插入成功 (ID: 2)
   ```

**檢查點：**
- ✅ 未完成的會話被記錄（completed=false）
- ✅ 任務的番茄鐘數量不變
- ✅ 計時器重置到 25:00

---

### 第四步：測試無關聯任務 📝

1. **直接進入計時器**
   - 不選擇任何任務
   - 直接在底部導航點擊計時器圖示

2. **開始並完成**
   - 點擊「開始」
   - 點擊「跳過」快速完成

3. **確認記錄**
   終端機應顯示：
   ```
   ✅ [TIMER] 番茄鐘會話已記錄到資料庫
      - 任務: 無關聯任務
      - 時長: 25 分鐘
      - 完成: true
   ```

**檢查點：**
- ✅ 會話正常記錄
- ✅ 終端機顯示「無關聯任務」
- ✅ 沒有錯誤訊息

---

## 📊 測試結果確認

### ✅ 測試通過的標誌：

1. **終端機日誌完整**
   - 每次開始都有「開始新的番茄鐘會話」
   - 每次完成/停止都有「番茄鐘會話已記錄到資料庫」
   - 資料庫插入成功訊息

2. **功能正常**
   - 完成的會話：任務番茄鐘數 +1
   - 未完成的會話：任務番茄鐘數不變
   - 計時器狀態正確切換

3. **無錯誤**
   - 沒有紅色錯誤訊息
   - 沒有異常中斷
   - 所有操作流暢
