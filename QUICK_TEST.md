# 快速測試 SQLite 整合

## 🚀 現在開始測試！

你的應用已經打開，現在按照以下步驟快速測試：

---

## 第一步：檢查終端機輸出 🔍

你應該在終端機（Terminal）看到類似的訊息：

### 如果這是全新安裝：
```
📚 [DB] 載入所有任務...
✅ [DB] 載入完成，共 0 個任務
```

### 如果有舊資料需要遷移：
```
🔄 需要執行資料遷移
🔄 開始資料遷移...
📦 發現 X 個任務需要遷移
✅ 成功遷移 X / X 個任務
💾 保留 SharedPreferences 中的資料作為備份
🎉 資料遷移完成！
📚 [DB] 載入所有任務...
✅ [DB] 載入完成，共 X 個任務
```

**請確認：** 你看到了哪種訊息？

---

## 第二步：測試任務新增 ➕

### 方法 1：使用 AI 助手（推薦）
1. 點擊右下角的「新增任務」按鈕（有 ✨ 圖示）
2. 在 AI 聊天介面輸入：
   ```
   我想學習 Flutter，需要花 3 個番茄鐘
   ```
3. 等待 AI 回覆
4. （目前需要手動創建任務，AI 自動提取功能稍後實現）

### 方法 2：手動創建（如果有編輯功能）
1. 手動創建一個測試任務
2. 設置標題、描述、番茄鐘數量、優先級

**終端機應該顯示：**
```
📝 [DB] 插入任務: [你的任務標題] (ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx)
✅ [DB] 任務插入成功
```

**檢查點：**
- ✅ 任務出現在「待辦事項」列表
- ✅ 終端機顯示插入訊息
- ✅ 任務資訊完整顯示

---

## 第三步：測試任務狀態切換 🔄

### 開始任務（Pending → In Progress）
1. 點擊任務卡片左側的 **圓圈圖示** ⭕
2. 觀察變化

**終端機應該顯示：**
```
✏️ [DB] 更新任務: [你的任務標題] (ID: xxxxx, 狀態: inProgress)
✅ [DB] 任務更新成功
```

**檢查點：**
- ✅ 任務移動到「進行中」區塊
- ✅ 圖示變成播放按鈕 ▶️
- ✅ 終端機顯示更新訊息

### 完成任務（In Progress → Completed）
1. 再次點擊任務的 **圖示**（現在是播放圖示）
2. 觀察變化

**終端機應該顯示：**
```
✏️ [DB] 更新任務: [你的任務標題] (ID: xxxxx, 狀態: completed)
✅ [DB] 任務更新成功
```

**檢查點：**
- ✅ 任務移動到「已完成」區塊
- ✅ 圖示變成勾選 ✓
- ✅ 任務標題有刪除線
- ✅ 任務卡片變半透明
- ✅ 終端機顯示更新訊息（包含 completed_at 時間戳）

---

## 第四步：測試任務編輯 ✏️

1. 點擊任務卡片（任何地方都可以）
2. 修改以下內容：
   - 更改標題（例如：添加 emoji 😊）
   - 修改描述
   - 調整番茄鐘數量
   - 更改優先級
3. 點擊「保存」

**終端機應該顯示：**
```
✏️ [DB] 更新任務: [新的任務標題] (ID: xxxxx, 狀態: xxxxx)
✅ [DB] 任務更新成功
```

**檢查點：**
- ✅ 所有修改立即反映在 UI 上
- ✅ 終端機顯示更新訊息

---

## 第五步：測試任務刪除 🗑️

1. 點擊任務卡片右上角的 **⋮** 選單
2. 選擇「刪除」
3. 在確認對話框中點擊「刪除」

**終端機應該顯示：**
```
🗑️ [DB] 刪除任務 ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
✅ [DB] 任務刪除成功
```

**檢查點：**
- ✅ 任務從列表中消失
- ✅ 終端機顯示刪除訊息
- ✅ 其他任務不受影響

---

## 第六步：測試資料持久化（最重要！）🔥

這是最關鍵的測試！

### 準備階段：
1. 創建 3 個測試任務：
   - **任務 A**：待辦（Pending）
   - **任務 B**：進行中（In Progress）
   - **任務 C**：已完成（Completed）
2. 記下這些任務的名稱和狀態

### 測試階段：
3. **完全關閉應用程式**
   - macOS: 按 `Command + Q`
   - 或者點擊紅色關閉按鈕，確保完全退出
4. 等待 3 秒
5. **重新啟動應用程式**
   - 重新執行 `flutter run -d macos`
   - 或者從 Xcode/VS Code 重新運行

### 驗證階段：
**終端機應該顯示：**
```
📚 [DB] 載入所有任務...
✅ [DB] 載入完成，共 3 個任務
```

**檢查點：**
- ✅ 所有 3 個任務都還在
- ✅ 任務 A 在「待辦事項」區塊
- ✅ 任務 B 在「進行中」區塊
- ✅ 任務 C 在「已完成」區塊
- ✅ 任務資訊完整（標題、描述、番茄鐘數、優先級）
- ✅ 任務狀態正確保留

---

## 第七步：測試番茄鐘整合 ⏱️

1. 點擊任一任務卡片上的 **⏱️ 計時器按鈕**
2. 觀察畫面切換

**檢查點：**
- ✅ 畫面切換到番茄鐘頁面
- ✅ 顯示 SnackBar：「已選擇任務：XXX」
- ✅ 番茄鐘頁面顯示正確的任務名稱
- ✅ 沒有任何錯誤訊息

## 💡 小提示

- **查看終端機日誌**：所有資料庫操作都會顯示 emoji 圖示（📝 ✏️ 🗑️ 📚）
- **確認完全關閉**：測試持久化時一定要完全退出應用程式
- **創建多個任務**：建議至少創建 3-5 個任務來測試
