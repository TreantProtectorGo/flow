# SQLite 整合測試指南

## 🎯 測試目標
驗證所有任務操作都正確使用 SQLite 資料庫，並且資料在應用重啟後能夠持久化。

---

## ✅ 測試檢查清單

### 1️⃣ 資料遷移測試（如果有舊資料）
- [ ] 查看終端機輸出，確認是否有遷移訊息：
  ```
  🔄 需要執行資料遷移
  📦 發現 X 個任務需要遷移
  ✅ 成功遷移 X / X 個任務
  🎉 資料遷移完成！
  ```
- [ ] 驗證舊任務是否出現在應用中
- [ ] 確認任務資料完整（標題、描述、番茄鐘數、優先級、狀態）

---

### 2️⃣ 任務新增測試
**步驟：**
1. 點擊右下角「新增任務」按鈕
2. 在 AI 聊天介面輸入任務需求，例如：
   ```
   我想寫一篇部落格文章，主題是 Flutter 狀態管理
   ```
3. AI 會回覆建議
4. （目前手動創建）或等待 AI 任務提取功能

**驗證：**
- [ ] 新任務出現在「待辦事項」列表
- [ ] 查看終端機輸出，確認有 `INSERT` SQL 語句
- [ ] 任務資訊完整顯示

---

### 3️⃣ 任務編輯測試
**步驟：**
1. 點擊任一任務卡片
2. 修改任務資訊：
   - 更改標題
   - 修改描述
   - 調整番茄鐘數量
   - 變更優先級
3. 點擊「保存」

**驗證：**
- [ ] 任務資訊立即更新
- [ ] 查看終端機輸出，確認有 `UPDATE` SQL 語句
- [ ] 所有修改都正確顯示

---

### 4️⃣ 任務狀態切換測試
**測試 A：開始任務（Pending → In Progress）**
- [ ] 點擊任務左側的圓圈圖示
- [ ] 任務移動到「進行中」區塊
- [ ] 圖示變成播放圖示（play_circle）
- [ ] 終端機顯示 `UPDATE` SQL 語句

**測試 B：完成任務（In Progress → Completed）**
- [ ] 再次點擊任務左側的圖示
- [ ] 任務移動到「已完成」區塊
- [ ] 圖示變成勾選圖示（check_circle）
- [ ] 任務標題顯示刪除線
- [ ] 任務卡片透明度降低
- [ ] 終端機顯示 `UPDATE` SQL 語句，包含 `completed_at`

**測試 C：取消完成（Completed → Pending）**
- [ ] 點擊已完成任務的圖示
- [ ] 任務返回「待辦事項」
- [ ] 刪除線消失
- [ ] 終端機顯示 `UPDATE` SQL 語句

---

### 5️⃣ 任務刪除測試
**步驟：**
1. 點擊任務卡片右上角的「⋮」選單
2. 選擇「刪除」
3. 確認刪除對話框

**驗證：**
- [ ] 任務從列表中消失
- [ ] 查看終端機輸出，確認有 `DELETE` SQL 語句
- [ ] 其他任務不受影響

---

### 6️⃣ 番茄鐘整合測試
**步驟：**
1. 點擊任務卡片上的「⏱️ 計時器」按鈕
2. 確認切換到番茄鐘畫面
3. 確認正確的任務已被選中
4. （可選）開始計時器

**驗證：**
- [ ] 當前任務 ID 正確設置
- [ ] SnackBar 顯示「已選擇任務：XXX」
- [ ] 計時器畫面顯示正確任務名稱
- [ ] 終端機沒有錯誤訊息

---

### 7️⃣ 資料持久化測試（最重要！）
**步驟：**
1. 創建 2-3 個測試任務，設置不同狀態：
   - 1 個待辦（Pending）
   - 1 個進行中（In Progress）
   - 1 個已完成（Completed）
2. 記下任務的詳細資訊
3. **完全關閉應用程式**（Command+Q）
4. 重新啟動應用程式

**驗證：**
- [ ] 所有任務都還在
- [ ] 任務狀態正確（待辦/進行中/已完成）
- [ ] 任務資訊完整（標題、描述、番茄鐘、優先級）
- [ ] 任務順序合理
- [ ] 當前任務 ID 保留（如果有）

---

### 8️⃣ 多任務壓力測試
**步驟：**
1. 快速創建 10+ 個任務
2. 批量修改多個任務的狀態
3. 刪除幾個任務
4. 編輯多個任務

**驗證：**
- [ ] 應用程式反應流暢，沒有卡頓
- [ ] 所有操作正確反映在 UI 上
- [ ] 終端機沒有錯誤或警告
- [ ] 資料庫操作正常執行

---

### 9️⃣ 錯誤處理測試
**步驟：**
1. 查看終端機輸出
2. 尋找任何錯誤訊息或警告

**驗證：**
- [ ] 沒有 SQL 錯誤
- [ ] 沒有空指針異常（Null check operator）

- [ ] 所有異步操作正常完成

## 🔍 檢查終端機輸出

### 預期的正常日誌：
```
✅ 資料庫初始化成功
✅ 沒有需要遷移的資料
或
🔄 開始資料遷移...
✅ 成功遷移 X / X 個任務
🎉 資料遷移完成！
```

### SQL 操作日誌（DatabaseHelper 的 debugPrint）：
```
📝 插入任務: [任務標題]
📝 更新任務: [任務 ID]
📝 刪除任務: [任務 ID]
📚 載入所有任務: [X] 個
```

### ⚠️ 需要注意的錯誤：
- `DatabaseException`: SQL 語法錯誤
- `type 'Null' is not a subtype of type`: 空值處理問題
- `Bad state: No element`: 列表查詢失敗
- `Unhandled Exception`: 未捕獲的異常

## 🐛 常見問題排查

### 問題 1：任務沒有持久化
**症狀：** 重啟後任務消失
**檢查：**
- 終端機是否有 `INSERT` 或 `UPDATE` SQL 語句？
- 是否有資料庫寫入錯誤？
- 資料庫文件路徑是否正確？

### 問題 2：資料遷移失敗
**症狀：** 舊任務沒有出現
**檢查：**
- 終端機是否顯示遷移訊息？
- 是否有 JSON 解析錯誤？
- SharedPreferences 中是否確實有舊資料？

### 問題 3：UI 更新延遲
**症狀：** 操作後 UI 沒有立即更新
**檢查：**
- 是否調用了 `notifyListeners()`？
- 資料庫操作是否使用了 `await`？
- `_tasks` 列表是否正確更新？

### 問題 4：任務重複
**症狀：** 同一個任務出現多次
**檢查：**
- 資料庫中是否有重複的 ID？
- 遷移過程是否執行了多次？
- `_tasks` 列表添加邏輯是否正確？
